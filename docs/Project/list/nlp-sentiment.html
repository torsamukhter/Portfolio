Overview

This project was completed as part of BAA1XXX: Advanced Analytics at Dublin City University. It compares two state-of-the-art AI models for sentiment classification of customer reviews:

  • BERT — a transformer-based discriminative model, fine-tuned on labelled data
  • GPT — a generative large language model, applied via zero-shot prompting

The business context is ShopEase, a medium-sized online clothing retailer operating in Ireland, which receives thousands of customer reviews monthly. Manual analysis is not scalable — NLP offers an automated, data-driven solution.

Note

The dataset used is the Women’s E-Commerce Clothing Reviews dataset from Kaggle, containing over 23,000 real customer reviews with ratings, department names, and review text.

Data Preprocessing

import re
from collections import Counter

def preprocess_text(text):
    """Clean and normalise review text."""
    text = text.lower()
    text = re.sub(r"[^a-z\s]", "", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text

# Apply preprocessing
df["Cleaned Review"] = df["Review Text"].apply(preprocess_text)

# Binary classification: Positive vs Negative only
df_binary = df[df["Sentiment"] != "Neutral"].copy()
df_binary["Label"] = df_binary["Sentiment"].map({"Positive": 1, "Negative": 0})

print(f"Binary dataset size: {len(df_binary)}")
print(f"Class balance:\n{df_binary['Sentiment'].value_counts()}")
print(f"\nExample cleaned review:\n{df_binary['Cleaned Review'].iloc[0][:200]}")

Most Common Words by Sentiment

Code 
stopwords = {"the","a","and","is","it","i","this","to","in","of","was",
             "for","my","so","but","on","with","at","be","have","had",
             "its","as","are","me","that","not","an","or","very","just",
             "would","did","which","very","all","what","from","they","were"}

def top_words(texts, n=15):
    words = []
    for t in texts:
        words.extend([w for w in t.split() if w not in stopwords and len(w) > 2])
    return Counter(words).most_common(n)

pos_words = top_words(df_binary[df_binary["Label"]==1]["Cleaned Review"])
neg_words = top_words(df_binary[df_binary["Label"]==0]["Cleaned Review"])

fig = make_subplots(rows=1, cols=2,
    subplot_titles=["Top Words — Positive Reviews", "Top Words — Negative Reviews"])

fig.add_trace(go.Bar(
    x=[w[1] for w in pos_words],
    y=[w[0] for w in pos_words],
    orientation="h",
    marker_color="#2ecc71",
    showlegend=False,
), row=1, col=1)

fig.add_trace(go.Bar(
    x=[w[1] for w in neg_words],
    y=[w[0] for w in neg_words],
    orientation="h",
    marker_color="#e74c3c",
    showlegend=False,
), row=1, col=2)

fig.update_layout(template="plotly_white", height=450,
    title="Most Frequent Words by Sentiment")
fig.show()

Methodology

The workflow followed a structured analytics pipeline:

  1. Problem Definition
    Identify the need for automated sentiment classification at scale.

  2. Data Preparation
    Clean and preprocess review text, handle missing values, and create sentiment labels.

  3. Exploratory Analysis
    Analyse rating distribution, review length, and department-level patterns.

  4. Model Implementation

    • Fine-tuned BERT on labelled data
    • Applied GPT using zero-shot prompting

  5. Evaluation
    Compared models using accuracy, precision, recall, and F1-score. ## Key Findings

ImportantResults Summary
  • BERT outperformed GPT across all metrics (F1: 0.91 vs 0.85)
  • BERT’s fine-tuning on domain-specific data gave it a significant edge
  • GPT’s zero-shot approach is more flexible but less precise on specialised tasks
  • Negative reviews tend to be longer — customers write more when dissatisfied
  • Dresses had the highest review volume; Jackets had the highest negative ratio

Ethical Considerations

WarningTrustworthy AI — 7 Requirements
  • Fairness: Models may be biased toward reviews from certain demographics (age, location)
  • Transparency: BERT’s fine-tuning process is more explainable than GPT’s black-box prompting
  • Privacy: Customer review data must comply with GDPR — ShopEase must ensure anonymisation
  • Accountability: Automated sentiment decisions should be auditable and human-reviewed
  • Robustness: Both models struggle with sarcasm and mixed-sentiment reviews

Business Recommendations

For ShopEase:

  1. Deploy BERT for production sentiment classification — higher accuracy justifies the compute cost
  2. Use GPT for exploratory analysis or when labelled data is unavailable
  3. Prioritise negative reviews flagged by the model for urgent customer service response
  4. Monitor model drift — retrain BERT quarterly as fashion language evolves
  5. Integrate with dashboard — pipe model outputs into a real-time Power BI or Plotly dashboard

Reflection

This project was the most technically challenging work I completed during my MSc. Fine-tuning a transformer model for the first time required understanding concepts I had only read about — tokenisation, attention mechanisms, and training loops. The comparison with GPT’s zero-shot approach taught me that the “best” model depends entirely on context: BERT wins on accuracy, but GPT wins on flexibility and cost when no labelled data exists. This trade-off is a genuinely important business decision, not just a technical one. ## Conclusion

This project demonstrated that BERT provides superior performance for sentiment classification when labelled data is available, while GPT offers flexibility in zero-shot scenarios. The choice between models depends on business constraints such as cost, data availability, and required accuracy.

From a business perspective, BERT is the recommended solution for ShopEase due to its higher accuracy and reliability in handling customer feedback at scale. However, GPT remains valuable for rapid prototyping and situations where labelled data is unavailable.

Overall, this analysis highlights the importance of aligning machine learning solutions with real-world business needs rather than purely technical performance.
